LoRabbit Privacy Policy

Last updated: 2026-04-11

Repository: https://github.com/Facta-Leopard/LoRabbit
Support URL: support.html

English

1. Scope

This privacy policy describes how the current public build of LoRabbit handles data when you use the app on your Mac.

2. Core privacy position

  • LoRabbit is designed for local-first use.
  • The app does not require a LoRabbit account.
  • The app does not upload selected models, datasets, training logs, or adapter outputs to a LoRabbit-operated server.
  • The app does not include in-app analytics, advertising SDKs, or tracking services in its current build.

3. Data processed on device

When you use LoRabbit, the app may process the following data locally on your device:

  • model directories and model metadata you choose
  • dataset files and folders you choose
  • training settings, compatibility reports, and preflight summaries
  • training logs, run summaries, and exported adapter artifacts

These items are processed and stored locally inside the active workspace that you choose, or inside the app’s default local workspace.

4. External links

If you open external pages such as project pages, support pages, or model reference pages, LoRabbit asks macOS to open those links in your default web browser. Those requests are handled by your browser and are subject to the browser’s own policies.

5. Data collection and tracking

  • LoRabbit does not intentionally collect personal data for advertising or analytics in the current public build.
  • LoRabbit does not use user data for tracking.
  • LoRabbit does not operate a cloud sync service for training sessions or training outputs.

6. User responsibility

You are responsible for ensuring that you have the right to use any model, dataset, or document that you load into the app.

You are also responsible for reviewing any third-party license, model card, or dataset terms that apply to the materials you use with LoRabbit.

7. Public privacy and support paths

  • Project repository: https://github.com/Facta-Leopard/LoRabbit
  • Support page: support.html
  • Support email: shteosis@gmail.com
  • Repository issues: https://github.com/Facta-Leopard/LoRabbit/issues
  • Repository discussions: https://github.com/Facta-Leopard/LoRabbit/discussions

8. Changes

If the app’s privacy practices change in a later public build, this policy may be updated.

한국어

1. 적용 범위

이 개인정보처리방침은 현재 공개 빌드 기준으로 LoRabbit가 Mac에서 데이터를 어떻게 다루는지 설명합니다.

2. 핵심 원칙

  • LoRabbit는 로컬 우선 사용을 전제로 설계되었습니다.
  • 앱은 별도의 LoRabbit 계정을 요구하지 않습니다.
  • 앱은 사용자가 선택한 모델, 데이터셋, 학습 로그, 어댑터 산출물을 LoRabbit 운영 서버로 업로드하지 않습니다.
  • 현재 공개 빌드에는 인앱 분석 SDK, 광고 SDK, 추적 서비스가 포함되어 있지 않습니다.

3. 기기 내에서 처리되는 데이터

LoRabbit를 사용할 때 앱은 아래 데이터를 사용자의 기기 안에서 로컬로 처리할 수 있습니다.

  • 사용자가 선택한 모델 디렉터리와 모델 메타데이터
  • 사용자가 선택한 데이터셋 파일과 폴더
  • 학습 설정, 호환성 보고서, preflight 요약
  • 학습 로그, 실행 요약, 어댑터 산출물

이 항목들은 사용자가 선택한 활성 워크스페이스 또는 앱의 기본 로컬 워크스페이스 안에 로컬로 처리·저장됩니다.

4. 외부 링크

프로젝트 페이지, 지원 문서, 모델 참고 페이지 같은 외부 링크를 열면, LoRabbit는 macOS를 통해 사용자의 기본 웹 브라우저로 해당 링크를 엽니다. 이 요청은 브라우저가 처리하며 브라우저 자체 정책이 적용됩니다.

5. 데이터 수집 및 추적

  • 현재 공개 빌드 기준으로 LoRabbit는 광고나 분석 목적으로 개인정보를 의도적으로 수집하지 않습니다.
  • LoRabbit는 사용자 데이터를 추적 목적으로 사용하지 않습니다.
  • LoRabbit는 학습 세션이나 학습 산출물을 위한 클라우드 동기화 서비스를 운영하지 않습니다.

6. 사용자 책임

사용자는 앱에 불러오는 모델, 데이터셋, 문서를 사용할 권리가 있는지 직접 확인해야 합니다.

또한 LoRabbit와 함께 사용하는 자료에 적용되는 제3자 라이선스, 모델 카드, 데이터셋 이용 조건도 직접 검토해야 합니다.

7. 공개 개인정보 및 지원 경로

  • 지원 페이지: support.html
  • 지원 이메일: shteosis@gmail.com
  • 저장소 이슈: https://github.com/Facta-Leopard/LoRabbit/issues
  • 저장소 디스커션: https://github.com/Facta-Leopard/LoRabbit/discussions

8. 변경

향후 공개 빌드에서 개인정보 처리 방식이 변경되면, 이 문서도 함께 업데이트될 수 있습니다.
