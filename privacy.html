English

1. Scope

This Privacy Policy describes how the current public build of LoRAbit handles data on your device.

2. Local-first operation

  • LoRAbit is designed for local, on-device use.
  • LoRAbit does not require a LoRAbit account.
  • LoRAbit does not operate a cloud sync service for sessions, training runs, or exported artifacts.
  • LoRAbit does not intentionally upload selected models, datasets, training logs, or adapter outputs to a LoRAbit-operated server.

3. Data processed on device

LoRAbit may process the following items locally on your device:

  • selected model directories and local model metadata
  • selected dataset files and folders
  • training settings, compatibility reports, and preflight summaries
  • training logs, run summaries, and exported local artifacts

These items are stored locally inside the active workspace you choose or inside the app’s default local workspace.

4. External links

If you open a model page, project page, support page, or other external URL from LoRAbit, the system opens that URL in your default web browser. Those requests are handled by your browser under the browser’s own policies.

5. Data collection and tracking

  • LoRAbit does not intentionally collect personal data for analytics or advertising in the current public build.
  • LoRAbit does not use personal data for tracking.
  • LoRAbit does not provide an in-app advertising, profiling, or account-based personalization system.

6. Your responsibility

You are responsible for confirming that you have the right to use any model, dataset, document, or other content that you load into LoRAbit.

You are also responsible for complying with any third-party license, model card, or usage restriction that applies to those materials.

7. Contact

  • Privacy and support email: shteosis@gmail.com
  • Project repository: Project repository / 프로젝트 저장소
  • Support page: support.html

8. Changes

If the app’s privacy practices change in a later public build, this page may be updated.

한국어

1. 적용 범위

이 개인정보처리방침은 현재 공개 빌드 기준으로 LoRAbit가 사용자의 기기에서 데이터를 어떻게 처리하는지 설명합니다.

2. 로컬 우선 동작

  • LoRAbit는 로컬 온디바이스 사용을 전제로 설계되었습니다.
  • LoRAbit는 별도의 LoRAbit 계정을 요구하지 않습니다.
  • LoRAbit는 세션, 학습 실행, 내보낸 산출물을 위한 클라우드 동기화 서비스를 운영하지 않습니다.
  • 현재 공개 빌드 기준으로 LoRAbit는 사용자가 선택한 모델, 데이터셋, 학습 로그, 어댑터 산출물을 LoRAbit 운영 서버로 의도적으로 업로드하지 않습니다.

3. 기기 안에서 처리되는 데이터

LoRAbit는 아래 항목을 사용자의 기기 안에서 로컬로 처리할 수 있습니다.

  • 선택한 모델 디렉터리와 로컬 모델 메타데이터
  • 선택한 데이터셋 파일과 폴더
  • 학습 설정, 호환성 보고서, preflight 요약
  • 학습 로그, 실행 요약, 내보낸 로컬 산출물

이 항목들은 사용자가 선택한 활성 워크스페이스 또는 앱의 기본 로컬 워크스페이스 안에 저장됩니다.

4. 외부 링크

LoRAbit에서 모델 페이지, 프로젝트 페이지, 지원 페이지 등 외부 URL을 열면 시스템이 해당 URL을 기본 웹 브라우저로 엽니다. 그 요청은 브라우저가 처리하며, 브라우저 자체 정책이 적용됩니다.

5. 데이터 수집 및 추적

  • 현재 공개 빌드 기준으로 LoRAbit는 분석이나 광고 목적으로 개인정보를 의도적으로 수집하지 않습니다.
  • LoRAbit는 개인정보를 추적 목적으로 사용하지 않습니다.
  • LoRAbit에는 인앱 광고, 프로파일링, 계정 기반 개인화 시스템이 없습니다.

6. 사용자 책임

사용자는 LoRAbit에 불러오는 모델, 데이터셋, 문서, 기타 콘텐츠를 사용할 권리가 있는지 직접 확인해야 합니다.

또한 해당 자료에 적용되는 제3자 라이선스, 모델 카드, 사용 제한도 직접 준수해야 합니다.

7. 연락처

  • 개인정보 및 지원 이메일: shteosis@gmail.com
  • 프로젝트 저장소: Project repository / 프로젝트 저장소
  • 지원 페이지: support.html

8. 변경

향후 공개 빌드에서 개인정보 처리 방식이 변경되면 이 페이지도 함께 업데이트될 수 있습니다.
